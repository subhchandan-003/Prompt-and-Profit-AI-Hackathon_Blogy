Section 01

Executive Summary & Objective

The Blogy AI Blog Engine is a browser-based, single-page application that takes a user-supplied seed keyword and systematically transforms it into a fully structured, SEO-optimised blog post through a five-stage AI prompt pipeline powered by the Anthropic Claude API.

The tool is designed specifically for the Prompt & Profit: BizMark'26 hackathon, fulfilling the requirements of Part 1 of the problem statement. It must survive a live stress-test by the judging panel, demonstrating repeatability, measurable SEO strength, and architectural clarity.

Core objective: Convert keyword intent → high-ranking, GEO-optimised, conversion-focused blog content through a structured, documented, and repeatable prompt flow — with every intermediate output visible to judges.

Section 02

Problem Statement

Content teams and startups using tools like Blogy face a recurring bottleneck: converting a raw keyword idea into a fully structured, SEO-validated blog post requires multiple disconnected tools, significant manual effort, and deep SEO expertise.

Current AI writing tools generate text in a single pass — with no keyword clustering, no SERP gap awareness, no structured prompt architecture, and no self-scored quality output. The result is generic content that doesn't rank.

What the Engine Solves

No keyword strategy → auto keyword clustering No competitive gap → SERP gap simulation No structure → AI-generated blog architecture No quality control → self-scoring SEO dashboard Not repeatable → documented prompt pipeline

Section 05

Five-Stage Pipeline Architecture

The engine operates as a sequential prompt pipeline. Each stage receives the outputs of all previous stages as context, building progressively richer intelligence before producing the final blog.

1
Keyword Intelligence & Clustering
Takes the seed keyword and expands it into a full keyword map. Identifies primary intent (informational / commercial / transactional), clusters secondary and LSI keywords, maps user questions, and estimates monthly search volume range and traffic potential.
Outputs
Primary keyword 5–8 secondary keywords LSI terms Search intent label Traffic estimate (low–high) Target audience profile
2
SERP Gap Analysis
Simulates what top-10 SERP results likely cover for this keyword, based on known ranking patterns. Then identifies 3–5 content gaps — angles, sub-topics, or depths of coverage that competitors consistently miss. These gaps form the differentiating hooks of the blog.
Outputs
Simulated competitor coverage map 3–5 identified SERP gaps Differentiating angle recommendation Featured snippet opportunity flag
3
Blog Architecture & Outline
Uses the keyword clusters and SERP gaps to generate a structured blog outline optimised for both ranking and conversion. Defines the H1 title, meta description, all H2/H3 sections with rationale, featured snippet target section, CTA placement, internal link anchor suggestions, and target word count.
Outputs
H1 title (keyword-optimised) Meta description (155 chars) Full H2/H3 outline Word count target Snippet-target section marked 3+ internal link anchors CTA placement recommendation
4
Full Blog Generation
Writes the complete blog using the Stage 3 outline, injecting keywords at the correct density (1–2% primary), using natural varied sentence structure to minimise AI detection, including the featured snippet section, internal link anchors, and a strong conversion CTA for Blogy. GEO optimisation signals (India-specific context) are embedded where relevant.
Outputs
Full blog (~1,500–2,000 words) Proper H1/H2/H3 formatting Keyword-dense but natural prose Featured snippet paragraph Internal link anchor text CTA block for Blogy
5
SEO Scorecard Generation
Self-evaluates the generated blog against all 10 judging metrics. Produces a scored rubric with a 0–100% score per metric, an overall SEO health score, and specific improvement flags. This scorecard is displayed as an animated dashboard in the UI.
Outputs
10 individual metric scores Overall SEO health score Keyword density % AI detection estimate % Naturalness score Snippet readiness % Improvement flags
Key architectural principle: Every stage passes its full output as context to the next stage. Stage 4 (blog generation) receives Stages 1–3 outputs simultaneously, ensuring perfect alignment between keyword strategy, SERP gaps, and content structure.

API Call Strategy

Each stage makes an independent API call with an accumulating system prompt context. Stage N+1 receives the full outputs of Stages 1 through N as user-turn context, ensuring each stage builds on prior intelligence.

API Key handling: The API key is entered once by the user in the UI and stored in memory (not localStorage). It is never hardcoded. The judges will use their own key or the team's key will be provided during demo.

Prompt Architecture Pattern

Every stage prompt follows this structure:

[System] You are an expert SEO strategist and content architect...
[Context] Seed keyword: {keyword}
[Prior outputs] Stage 1 output: {json}
[Task] Perform Stage 2: SERP gap analysis. Output in the following JSON structure: {...}
[Constraints] Do not include preamble. Respond only in the specified format.

SEO Scorecard — Metric Definitions

Stage 5 scores the blog against 10 metrics. Each metric is scored 0–100 by the AI rubric. The definitions and target thresholds are specified below.

Prompt Architecture Clarity
Is each stage prompt clearly structured and purposeful?
Target: 90%
Keyword Clustering Logic
Are primary, secondary, and LSI keywords properly identified?
Target: 85%
SERP Gap Identification
Are meaningful content gaps identified vs competitor pages?
Target: 80%
Projected Traffic Potential
Is a credible monthly search volume estimate provided?
Target: 75%
SEO Optimisation Percentage
Overall on-page SEO score via rubric (title, meta, H-tags, density)
Target: 80%
AI Detection Estimate
Lower is better — estimates likelihood of AI detection (inverted score)
Target: <30%
Naturalness Score
Does the writing read like a human expert wrote it?
Target: 85%
Snippet Readiness Probability
Does a section directly answer a likely featured snippet query?
Target: 75%
Keyword Density Compliance
Is primary keyword density between 1–2%? Secondary at 0.5–1%?
Target: 80%
Internal Linking Logic
Are 3+ relevant internal link anchors naturally embedded?
Target: 80%
